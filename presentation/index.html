The Bug

  • Occurs in Firefox's JIT (Just-In-Time) compiler IonMonkey
  • Exact details of the bug are still hidden from the public
  • Incorrect aliasing of array elements can lead to a Type Confusion bug

What is a JIT engine?

  • Turns managed code into native code as it runs
  • Analyzes code and performs optimizations

The Bug

  • Below is IonMonkey's pipeline architecture
  • Based on the limited patch notes, the bug most likely occurs during the "alias analysis" optimization step

Type Confusion Bugs

  • What is a type confusion bug?
  • They occur when an object of 1 type is backed in memory by an object of another type
  • How?
    • Constructor overwritten
Depending on the type of objects, r/w of pointers can occur

Exploitation

  • Could be used to bypass ASLR (Address Space Layout Randomization) by determing the address of the vtable (Virtual method table)
    • ASLR randomizes the base address of an executable when it's loaded into memory
    • The vtable contains pointers to function definitions
Object's member overlaps the vtable for BitmapData

Code Execution

  • ASLR bypassed and arbitrary r/w primitive
