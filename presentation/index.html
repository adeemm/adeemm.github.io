Perilous Peripherals

New Firmware Research

Presentation by: Adeem Mawani

Background Info

  • New resarch by Jesse Michael and Rick Altherr of Eclypsium
  • Analyzes the prevalance of unsigned firmware updates on peripheral devices

What is firmware signing?

  • Firmware governs the low-level behavior of a component
  • It may be burned into the integrated circuit of the device, or stored in flash memory
  • Signing allows for verification via cryptographic keys before running the code. This validates that the firmware is authentic and can be trusted

The Research

  • Found many devices accept unsigned firmware updates
  • These devices include: WiFi Adapters, USB Hubs, Trackpads, Laptop Cameras And Network Interface Cards
  • From Lenovo, Dell, HP and other major manufacturers

Compromise

  1. An attacker gains access to a device via any method, such as malware delivered via email
  2. With basic user privileges, the attacker writes malicious firmware to a vulnerable component
  3. If the component doesn’t require the firmware to be properly signed, the attacker’s code is loaded and run by the component
  4. The attacker can then use the unique functionality and privileges of that component to further an attack

Special Privileges

  • Firmware on a network adapter could allow an attacker to sniff, copy, redirect, block, or alter traffic (eg. block logging or alerts)
  • A compromised hard drive could allow the attacker to hide code and tools without being seen by the OS
  • PCI-based devices could enable Direct Memory Access attacks (direct R/W of memory, bypassing the OS and CPU)

Implications

  • Malware stays undetected by any software security controls
  • In many cases, doesn't even require priviledge escalation to a root user
  • (some firmware updates can be installed with average user permissions)

Fixes?

  • Despite previous in-the-wild attacks, peripheral manufacturers have been slow to adopt the practice of signing firmware
  • Even from some PoC's demonstrated, these vendors still have no plans to implement firmware signing
