Sudo Vulnerability

Another one...

Presentation by: Adeem Mawani

Background Info

  • Found by Apple security researcher Joe Vennix
  • Tracked as CVE-2019-18634
  • Allows for priviledge escalation

Previous Vulnerability

  • CVE-2019-14287 discovered back in October 2019
  • Also found by Vennix
  • Allowed arbitrary command execution as root, even in cases where root access is disallowed

This Vulnerability

  • Affects Sudo versions prior too 1.8.26
  • Can only be exploited when the "pwfeedback" option is enabled
    • This option gives visual feedback (*) when typing your password

How It Works

  • Stack-based buffer overflow
  • During a write error, the function erasing the asterisks (pwfeedback) doesn't properly reset the buffer position, but it does reset the remaining buffer length
  • Allows for writing past end of the buffer

PoC


						$ perl -e 'print(("A" x 100 . "\x{00}") x 50)' | sudo -S id

The Good News

  • This option is disabled by default in most distros (except Linux Mint & Elementary OS)
