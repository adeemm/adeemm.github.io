OpenSMTPD Vulnerabilities

Presentation by: Adeem Mawani

Background Info

  • New resarch by Qualys researchers
  • Found 2 vulnerabilities in OpenSMTPD
  • Fix released on Feb 24th

First Vulnerability

  • Lower impact
  • Allows local users to read the contents of arbitrary files
  • Read the first line of an arbitrary file or entire contents of another user's file

First Vulnerability Cont.

  • Generally not exploitable on most Linux distributions
  • Due to /proc/sys/fs/protected_hardlinks being set to "1" by default

Second Vulnerability

  • Out-of-bounds read flaw
  • Allows execution of arbitrary shell commands as root
  • Resides in client-side code flow

Exploitation Steps

  • For remote exploitation, send mail that creates a bounce
  • Next, OpenSMTPD connects back to their mail server to deliver the bounce
  • This allows for remote exploitation of OpenSMTPDâ€™s client-side vulnerability
  • However, for shell commands to be executed, the attacker must crash OpenSMTPD and wait until it's restarted
